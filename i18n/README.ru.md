## ▶️ Использование

Основной рабочий процесс:

1. Редактируйте `README.md` (канонический английский/корневой контент профиля).
2. Поддерживайте языковые файлы в `i18n/` в синхронизированном состоянии.
3. Проверяйте ссылки и разделы.
4. При публикации README-обновлений коммитьте только `README.md`.

Скрипт-помощник для push только README:

```bash
scripts/push_readme_only.sh -m "Update profile README" -b main
```

Поведение скрипта:

1. Выполняет fetch из `origin`.
2. Создает временный worktree из `origin/<branch>`.
3. Копирует корневой `README.md` во временный worktree.
4. Выполняет commit и push только `README.md`.
5. Очищает временное состояние.

## ⚙️ Конфигурация

- Языковые опции сохраняются как **одна строка выбора языка вверху** каждого варианта README.
- Ссылки в корневом README сейчас указывают на:
  - `i18n/README.ar.md`, `README.es.md`, `README.fr.md`, `README.ja.md`, `README.ko.md`, `README.vi.md`, `README.zh-Hans.md`, `README.zh-Hant.md`, `README.de.md`, `README.ru.md`
- Ссылки на спонсорство настроены в `.github/FUNDING.yml`.

## 🧯 Устранение неполадок

- `No README changes compared to origin/main`: локальный `README.md` совпадает с удаленным; push не требуется.
- Некорректное отображение изображений/баннера: проверьте пути вроде `figs/banner.png` и GitHub raw/blob URL.
- Несогласованные строки навигации по языкам: сохраняйте ровно одну строку языковых опций вверху каждого варианта README.
- Расхождения между корневым и i18n-контентом: обновляйте переводы после финализации правок корневого README.
